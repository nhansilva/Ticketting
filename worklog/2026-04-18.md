## Bugs fixed

### Bug 1: ReactiveRedisTemplate bean ambiguity (@Qualifier thiếu trong runtime)
- **Lỗi:** `Parameter 0 of method eventCacheStrategy required a single bean, but 2 were found: reactiveRedisTemplate, reactiveStringRedisTemplate`
- **Nguyên nhân:** Mặc dù đã fix trong code, class cũ trong `target/` chưa bị xoá → cần `mvn clean compile` trước khi run
- **Fix:** `mvn clean compile -pl event-catalog-service -am -DskipTests` để force recompile toàn bộ
- **Note:** Luôn clean trước khi run sau khi sửa `@Configuration` class

### Bug 2: MongoDB Unauthorized — `Command createIndexes requires authentication`
- **Lỗi:** `Command failed with error 13 (Unauthorized): 'Command createIndexes requires authentication'`
- **Nguyên nhân:** `application.yml` dùng URI `mongodb://localhost:27017/ticketing_catalog` không có credentials, nhưng MongoDB container được cấu hình với auth (`MONGO_INITDB_ROOT_USERNAME: ticketing`)
- **Fix:** Cập nhật `application.yml`:
  ```yaml
  uri: ${MONGODB_URI:mongodb://ticketing:ticketing123@localhost:27017/ticketing_catalog?authSource=admin}
  ```
- **Note:** Mọi service dùng MongoDB phải có credentials trong URI. `authSource=admin` vì root user được tạo trong db `admin`

### Bug 3: Docker Desktop không chạy — MongoDB/Redis port refused
- **Lỗi:** `Connection refused: getsockopt: localhost/[::1]:27017` và `localhost/127.0.0.1:6379`
- **Nguyên nhân:** Docker Desktop bị tắt trong quá trình session, containers không còn running
- **Fix:** Khởi động lại Docker Desktop → `docker compose up -d`
- **Note:** Kiểm tra Docker trước khi run service bằng `powershell.exe -Command "docker ps"`

### Bug 4: `@RequestParam` thiếu explicit name — `parameter name information not available`
- **Lỗi:** `Name for argument of type [EventType] not specified... Ensure that the compiler uses the '-parameters' flag`
- **Nguyên nhân:** Spring WebFlux 6.x yêu cầu tên tham số explicit khi không có `-parameters` compiler flag. `@RequestParam(required = false) EventType type` không đủ — phải có `name = "type"`
- **Fix:** 
  1. Thêm `<parameters>true</parameters>` vào `maven-compiler-plugin` trong root `pom.xml` (fix permanent cho toàn project)
  2. Add explicit `name` cho tất cả `@RequestParam` trong EventController, VenueController, SeatController:
  ```java
  // Trước
  @RequestParam(required = false) EventType type
  // Sau
  @RequestParam(name = "type", required = false) EventType type
  ```
- **Note:** **Áp dụng cho tất cả service sau này** — luôn dùng `name = "..."` với `@RequestParam`, không dựa vào parameter name reflection
